# DevSwitch UI 原型文档

创建日期：2026-06-08

## 1. 前端实现方式

DevSwitch 前端采用 Windows 原生 GUI 技术栈：

```text
C# + WinUI 3 + XAML + MVVM
```

实现约定：

- XAML 负责页面结构、控件、样式和数据绑定。
- C# ViewModel 负责 SDK 列表、下载任务、doctor 结果和按钮命令。
- 使用 WinUI 3 `NavigationView` 实现左侧分类导航。
- 使用 WinUI 3 `ListView` + `Grid` 行布局实现版本列表。
- 使用 `ContentDialog` 实现导入、下载、删除、重置等弹窗。
- 使用 `.resw` 资源文件做简体中文和 English 文案。
- 不使用 Electron、WebView、HTML/CSS/JavaScript 做主界面。

## 2. 主界面

### 2.1 左侧分类

```text
Java
Maven
Node.js
Go
```

行为：

- 点击分类后右侧展示该 SDK 类型的版本列表。
- 当前分类高亮。
- 如果某分类没有已登记版本，显示空状态引导。

### 2.2 顶部操作区

```text
[新增本地 SDK] [下载 SDK] [重置] [Doctor]
```

- 新增本地 SDK：打开导入弹窗。
- 下载 SDK：打开下载弹窗。
- 重置：恢复工具自身环境配置。
- Doctor：进入诊断页面。

### 2.3 版本表格

字段：

```text
名称 / 版本 / 来源 / 路径 / 状态 / 操作
```

来源：

```text
外部
托管
```

状态：

```text
使用中
可用
不可用
未验证
```

操作按钮：

```text
切换
验证
编辑
删除
```

当版本处于使用中时：

- 显示绿色勾或高亮标签。
- “切换”按钮禁用或变成“当前”。

当版本不可用时：

- 显示红色状态。
- “切换”按钮禁用。
- 提供“查看原因”。

## 3. 空状态

当某分类没有 SDK 时：

```text
当前还没有 Java 版本

你可以导入本机已有的 JDK 根目录，或从版本源下载一个 JDK。

[新增本地 SDK] [下载 SDK]
```

## 4. 新增本地 SDK 弹窗

```text
┌──────────────────────── 新增本地 SDK ────────────────────────┐
│ SDK 类型： Java v                                             │
│ SDK 根目录： [D:\Programs\java\jdk-17.0.10        ] [浏览] │
│ 名称：      [Temurin 17                                  ]    │
│                                                              │
│ 检测结果：                                                    │
│ ✓ 找到 bin\java.exe                                          │
│ ✓ 找到 bin\javac.exe                                         │
│ ✓ 识别版本：17.0.10                                           │
│                                                              │
│ 注意：请选择 SDK 根目录，不要选择 bin 目录。                  │
│                                                              │
│                                      [取消] [导入]            │
└──────────────────────────────────────────────────────────────┘
```

行为：

- 用户选择目录后立即轻量检测。
- 如果误选 `bin`，提示并自动建议父目录。
- 名称默认自动生成，允许用户编辑。
- 导入外部 SDK 时只登记路径，不复制文件。


只允许编辑：

- 名称。

- SDK 类型。
- 真实版本，真实版本由版本识别结果决定。

## 7. 删除确认

外部 SDK：

```text
确认删除这个 SDK 记录？

这只会从 DevSwitch 列表中移除记录，不会删除磁盘上的 SDK 文件。

[取消] [删除记录]
```

托管 SDK：

```text
确认删除这个托管 SDK？

可以只删除记录，也可以同时删除 DevSwitch 下载并托管的 SDK 文件。

[取消] [只删记录] [删除记录和文件]
```

## 11. 错误展示

Info：

```text
Java 已切换到 Temurin 17。请重启已打开的终端。
```

Warning：

```text
已切换，但检测到 PATH 中存在更靠前的旧 Java 路径。请前往 Doctor 查看建议。
```

Error：

```text
切换失败，已自动回滚到原版本。原因：目标 SDK 缺少 bin\java.exe。
```

Fatal：

```text
配置迁移失败，DevSwitch 已保留旧配置。

[打开配置目录] [导出诊断包]
```
